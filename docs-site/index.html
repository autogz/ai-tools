FAQ

What does "free repo diagnosis" include?

A detailed report analyzing your GitHub repo: README quality, missing documentation files (LICENSE, SECURITY, CONTRIBUTING), CI/CD presence, PyPI readiness, and 5 specific improvement suggestions with effort estimates.

How is this different from other code generators?

Most tools generate code. AI Tools generates everything around your code — the README, docs, CI config, license, and launch infrastructure. It's like having a developer relations engineer on your team.

Do I need an API key?

Yes. The diagnosis and generation features use AI models. You need an OpenAI API key (or compatible provider). Set it once via aitools config or the OPENAI_API_KEY environment variable.

How does payment work?

Send USDT (ERC20) to the wallet address shown above. Then run aitools claim <tx_hash>. The CLI queries Etherscan to verify the transaction and generates a local activation key. No server involved, no data collected.

What if I send USDT on the wrong network?

Only ERC20 (Ethereum) is supported. Transactions on BSC, Solana, TRC20, or other networks will be lost. Always verify the network and address before sending.

Is my payment refundable?

All blockchain transactions are irreversible. Please verify the address and network before sending. That said, if there's a genuine error (e.g., you sent double and contact us), we work it out case by case.

Do you collect my data?

No telemetry, no analytics, no tracking. All tools run locally. Your code, API keys, and data stay on your machine. The only network requests are to your configured AI provider and Etherscan (for license activation).

What if I need help?

Open an issue on GitHub or check the documentation. We're a solo operation, so response times may vary, but we do respond.
